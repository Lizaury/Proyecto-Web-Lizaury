1.- Atributos globales

1.1 class

Se utiliza para aplicar estilos CSS o seleccionar elementos en JavaScript. Es clave para crear diseños dinámicos y reutilizables, ya que permite agrupar elementos con características similares.

1.2 id

Identifica de manera única un elemento en el documento. Es fundamental para la navegación interna de la página, manipulación directa en JavaScript, y estilos CSS específicos.

1.3 style

Permite aplicar estilos en línea directamente al elemento. Útil para estilos rápidos o únicos, aunque no se recomienda para proyectos grandes por falta de modularidad.

1.4 title

Proporciona una descripción adicional que se muestra como un tooltip cuando el usuario pasa el cursor sobre el elemento. Mejora la accesibilidad y brinda contexto adicional a los usuarios.

1.5 lang

Define el idioma del contenido dentro del elemento. Mejora la accesibilidad y ayuda a los motores de búsqueda a interpretar correctamente el idioma del contenido.

2.- Atributos form

2.1 action

Define la URL donde se enviarán los datos del formulario, lo que determina el destino del procesamiento de la información.

2.2 method

Define el método HTTP que se usará para enviar los datos. Sus valores comunes son GET (Los datos se envían en la URL) y POST (Los datos se envían en el cuerpo de la solicitud. Es más seguro para enviar información sensible).

2.3 enctype

Especifica cómo se deben codificar los datos enviados. Es crucial para formularios que necesitan subir archivos o manejar datos en formatos específicos.

2.4 target

Determina dónde se abrirá la respuesta del servidor tras enviar el formulario. Controla dónde se abrirá la respuesta al enviar el formulario, permitiendo mayor flexibilidad en la interacción del usuario.

2.5 autocomplete

Controla si el navegador debe autocompletar los campos del formulario basándose en datos previos del usuario. Sus valores posibles son on, que habilita el autocompletado (por defecto), y off, que desactiva el autocompletado.

3.- Atrbutos input

3.1 type

Define la naturaleza del campo de entrada, permitiendo capturar distintos tipos de datos como texto, correos electrónicos, contraseñas, números, archivos, etc. Es fundamental para crear formularios funcionales y adaptados a las necesidades del usuario.

Valores comunes:

  • text: Entrada de texto simple.
  • password: Entrada oculta para contraseñas.
  • email: Entrada de correos electrónicos.
  • number: Números enteros o decimales.
  • file: Para subir archivos.
  • checkbox, radio: Botones de selección.
  • submit: Botón para enviar el formulario.

3.2 name

Identifica el campo al enviar datos al servidor, actuando como clave para los valores ingresados. Sin este atributo, los datos no pueden ser procesados correctamente.

3.3 value

Define un valor predeterminado que se muestra en el campo al cargar la página, útil para datos predefinidos o para botones de envío.

3.4 placeholder

Muestra un texto de ejemplo dentro del campo, útil como guía para el usuario. Esto mejora la experiencia del usuario mostrando texto guía dentro del campo, indicando qué tipo de información debe ingresar.

3.5 required

Obliga al usuario a completar el campo antes de enviar el formulario, ayudando a asegurar que se recopile la información esencial.
