1. -Etiquetas de Documento

1.1 html

Es la etiqueta raíz que envuelve todo el contenido de un documento HTML. Todo el código HTML debe ir dentro de esta etiqueta, lo que indica al navegador que se trata de un documento HTML.

1.2 head

La etiqueta <head> contiene metadatos (información sobre el documento) que no se muestran directamente en la página. Esto incluye el título de la página (<title>), enlaces a hojas de estilo externas (<link>), metadatos sobre el documento (<meta>), y scripts (<script>), entre otros.

1.3 body

La etiqueta <body> contiene todo el contenido visible de la página web, como textos, imágenes, enlaces, formularios, etc. Es lo que se muestra al usuario en el navegador.

2.- Etiquetas de Meta Datos

2.1 meta

Proporciona metadatos sobre el documento HTML. No tiene contenido visible y se utiliza en la sección <head> para especificar información como la codificación de caracteres, descripción, palabras clave para SEO, y configuraciones del navegador.

2.2 style

Contiene reglas CSS que definen el estilo de los elementos HTML en la página. Generalmente se usa en la sección <head>, aunque su uso es menos común cuando se utilizan hojas de estilo externas.

2.3 title

Define el título de la página que aparece en la pestaña del navegador o en los resultados de los motores de búsqueda. Debe colocarse dentro de la etiqueta <head>.

2.4 base

Establece la URL base para todas las URL relativas de la página. Permite definir un punto de referencia para enlaces e imágenes. Solo puede haber una etiqueta <base> por documento y debe colocarse dentro del <head>.

2.5 link

Se utiliza para vincular recursos externos, como hojas de estilo CSS, iconos o feeds. Es una etiqueta vacía (no tiene contenido) y se coloca dentro del <head>.

3.- Etiquetas de Agrupacion

3.1 p

Representa un párrafo de texto. Se utiliza para agrupar contenido textual. El navegador añade automáticamente un espacio antes y después de cada párrafo.

3.2 hr

Crea una línea horizontal que sirve para dividir contenido visualmente. Es una etiqueta vacía (sin contenido) y puede personalizarse con CSS.

3.3 ul

Representa una lista desordenada. Sus elementos se definen con la etiqueta <li> y suelen estar marcados con viñetas (aunque esto se puede personalizar).

3.4 div

Es un contenedor genérico que no tiene significado semántico propio. Se utiliza para agrupar otros elementos y aplicar estilos o scripts a través de CSS o JavaScript.

3.5 figure

Representa contenido ilustrativo, como imágenes, diagramas o gráficos. Se utiliza junto con <figcaption> para proporcionar una descripción o título del contenido visual.

3.6 figcaption

Proporciona una descripción o leyenda para el contenido de un <figure>. Debe estar directamente dentro de la etiqueta <figure>.

4.- Etiquetas Textuales

4.1 a

Define un hipervínculo que permite enlazar a otra página, sección del documento o recurso. Se usa junto con el atributo href para especificar la URL de destino. La etiqueta <a> no solo sirve para crear enlaces básicos, sino que puede incluir atributos adicionales como target, download, y eventos JavaScript (onclick) para extender su funcionalidad.

4.2 span

Es un contenedor en línea y genérico sin significado semántico propio. Se utiliza para aplicar estilos o scripts a partes específicas de texto o elementos en línea. El <span> se utiliza en texto en línea para aplicar estilos, scripts o añadir clases específicas. Es muy útil para manejar partes específicas de un texto sin romper su flujo.

4.3 sub

Representa texto en subíndice, es decir, texto que aparece más pequeño y ligeramente por debajo de la línea base. Es comúnmente usado en fórmulas matemáticas o químicas. El sub no solo se usa en fórmulas químicas o matemáticas, también tiene aplicaciones en contenido técnico y anotaciones.

4.4 sup

Representa texto en superíndice, que aparece más pequeño y elevado por encima de la línea base. Se usa frecuentemente en exponentes o notas al pie. El <sup> es útil para números de notas al pie, exponentes o fórmulas científicas. También puede combinarse con estilos personalizados.

4.5 cite

Indica el título de una obra o referencia. Se utiliza para resaltar el nombre de un libro, película, artículo, etc., generalmente con formato en cursiva (esto depende del navegador). Puede combinarse con atributos como title y personalizarse para proporcionar contexto adicional.

5.-Etiquetas Semanticas

5.1 nav

La etiqueta <nav> define una sección específica destinada a la navegación dentro del documento o sitio web. Contiene enlaces que permiten a los usuarios moverse entre diferentes secciones del sitio (como menús, índices o tablas de contenido). Es semánticamente importante, ya que los lectores de pantalla la identifican como una zona de navegación.

5.2 section

La etiqueta <section> agrupa contenido relacionado dentro de un documento. Cada sección debe tener un propósito claro, como representar un tema específico o una división lógica en la página. Está diseñada para mejorar la organización del contenido y facilitar la lectura tanto para los usuarios como para los motores de búsqueda.

5.3 article

La etiqueta <article> representa una pieza de contenido independiente que tiene sentido por sí misma. Su objetivo es estructurar contenido que pueda ser reutilizado o compartido de manera autónoma, como artículos de blogs, comentarios, publicaciones en foros, o noticias.

5.4 header

La etiqueta <header> define la cabecera de un documento, sección o artículo. Su propósito es proporcionar información introductoria o elementos de navegación relacionados con el contenido.

5.5 main

La etiqueta <main> representa el contenido central de una página web que es único y relevante. Ayuda a los lectores de pantalla a saltar directamente al contenido principal, mejorando la accesibilidad.

5.6 footer

La etiqueta <footer> representa el pie de página de un documento, sección o artículo. Generalmente contiene información complementaria, como derechos de autor, enlaces secundarios, o datos de contacto.
